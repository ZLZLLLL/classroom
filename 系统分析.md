## 3.4 系统功能需求分析

### 3.4.1 学生用例分析

## 3.5 系统的主要业务流程分析

### 3.5.1 签到流程

签到是课堂互动的核心功能之一，用于记录学生的课堂出勤情况。

**签到流程说明：**

1. **教师发起签到**
   - 教师选择需要签到的课程
   - 设置签到时长（默认15分钟，可选1-60分钟）
   - 可设置签到位置（可选，用于定位签到）
   - 可选择针对特定班级（可选，不选则对所有选课学生生效）
   - 点击"发起签到"按钮

2. **系统处理**
   - 系统创建签到活动记录
   - 设置签到结束时间
   - 向选课学生推送签到通知

3. **学生签到**
   - 学生收到签到通知
   - 进入签到页面，查看进行中的签到活动
   - 点击"签到"按钮提交签到
   - 如设置了位置，系统验证签到位置

4. **签到结果**
   - 系统记录签到时间
   - 签到成功，返回成功提示
   - 教师可实时查看签到统计（已签到/未签到名单）

### 3.5.3 作业流程

作业流程包括发布、提交和批改三个主要阶段。

**作业流程说明：**

1. **发布作业**
   - 教师选择课程
   - 填写作业标题、描述
   - 设置截止时间
   - 可添加附件要求
   - 点击"发布作业"

2. **系统处理**
   - 创建作业记录
   - 关联课程和班级
   - 向选课学生推送作业通知

3. **学生提交**
   - 学生收到作业通知
   - 查看作业详情
   - 填写作业答案或上传附件
   - 在截止时间前提交作业

4. **教师批改**
   - 教师查看学生提交的作业
   - 审阅作业内容
   - 给出分数和评语
   - 提交批改结果

5. **成绩反馈**
   - 系统更新作业成绩
   - 学生收到批改通知
   - 学生查看作业成绩和教师评语

| 回答 | Answer | 回答ID、问题ID、回答用户ID、内容、创建时间 |
| 点赞 | Like | 点赞ID、用户ID、目标ID、目标类型、创建时间 |
| 作业 | Homework | 作业ID、课程ID、标题、描述、截止时间、创建时间 |
| 作业提交 | HomeworkSubmit | 提交ID、作业ID、用户ID、提交内容、提交时间、分数、评语 |
| 积分 | Points | 积分ID、用户ID、课程ID、积分值、类型、描述、创建时间 |

### 3.6.4 数据流程说明

**用户管理数据流：**
1. 用户注册 → 填写基本信息 → 存储到User表
2. 用户登录 → 验证账号密码 → 生成Token返回
3. 管理员操作 → CRUD用户信息 → 更新User表

**课程管理数据流：**
1. 教师创建课程 → 填写课程信息 → 存储到Course表
2. 学生选课 → 关联学生与课程 → 存储到CourseStudent表
3. 查询课程 → 从Course表获取数据 → 返回给用户

**课堂互动数据流：**
1. 教师发起签到 → 创建Attendance记录 → 推送通知给学生
2. 学生签到 → 创建AttendanceRecord → 更新统计数据
3. 学生提问 → 创建Question记录 → 显示在问题列表
4. 教师回答 → 创建Answer记录 → 关联到问题

**作业管理数据流：**
1. 教师发布作业 → 创建Homework记录 → 通知选课学生
2. 学生提交 → 创建HomeworkSubmit记录 → 状态变为"已提交"
3. 教师批改 → 更新HomeworkSubmit的分数和评语 → 通知学生

**积分管理数据流：**
1. 触发积分动作（签到、提问、提交作业） → 记录Points表
2. 查询积分 → 聚合用户的所有积分记录 → 计算总积分
3. 积分排行 → 按课程分组统计 → 排序返回
